# TrendX Design Reference Catalog

Source: [VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md)

This catalog intentionally keeps only the references most useful for TrendX.

## Primary References

### Coinbase

- Files:
  - [DESIGN.md](./coinbase/DESIGN.md)
  - [README.md](./coinbase/README.md)
- Use when:
  - the page should feel trustworthy and financial
  - the layout should stay clear under real data density
  - you want a clean exchange-grade visual baseline

### Kraken

- Files:
  - [DESIGN.md](./kraken/DESIGN.md)
  - [README.md](./kraken/README.md)
- Use when:
  - the page should feel closer to a trading terminal
  - dark surfaces and dense signal cards are appropriate
  - the dashboard needs more tactical energy than Coinbase

### Revolut

- Files:
  - [DESIGN.md](./revolut/DESIGN.md)
  - [README.md](./revolut/README.md)
- Use when:
  - you want premium fintech polish
  - glassy panels and strong hierarchy fit the page
  - account and portfolio views need a higher-end feel

## Secondary References

### Cohere

- Files:
  - [DESIGN.md](./cohere/DESIGN.md)
  - [README.md](./cohere/README.md)
- Use when:
  - product storytelling and data visualization need more polish
  - the dashboard should feel modern without looking generic

### Framer

- Files:
  - [DESIGN.md](./framer/DESIGN.md)
  - [README.md](./framer/README.md)
- Use when:
  - you need hero composition ideas
  - you want stronger motion or landing-page rhythm
  - you are polishing marketing sections, not core trading panels

## Recommended Pairings For TrendX

- `coinbase` + `cohere`
  - Best for a clean, operator-friendly first dashboard.
- `kraken` + `revolut`
  - Best for a more aggressive dark trading console.
- `coinbase` + `framer`
  - Best for landing page first, dashboard second.

## Working Rule

Choose the page structure and component hierarchy first, then apply one primary reference and at most one secondary reference.
